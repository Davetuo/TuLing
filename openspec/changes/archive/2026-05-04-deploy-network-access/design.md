## Context

当前项目后端 NestJS 已绑定 `0.0.0.0`，但 CORS origin 硬编码为 `http://localhost:5173`；前端 Vite 默认只绑定 localhost。要在局域网其他设备（手机、平板等）上访问服务，需要：

1. 前端 Vite 绑定所有网络接口 (`0.0.0.0`)
2. 后端 CORS 接受来自其他设备的请求源（如 `http://192.168.x.x:5173`）
3. 提供生产模式：构建前端静态文件，由后端统一托管（减少跨域问题）

约束：开发模式下前后端分离运行（热重载），生产模式下后端托管前端静态文件（单端口服务）。

## Goals / Non-Goals

**Goals:**
- 开发模式下，前端 Vite 绑定 `0.0.0.0`，其他设备可通过 `http://<host-ip>:5173` 访问
- 后端 CORS 通过环境变量 `CORS_ORIGINS` 配置，支持逗号分隔的多来源
- 生产模式下，`npm run build` 构建前端，NestJS 托管 `client/dist/` 静态文件
- 提供生产模式启动脚本（构建前端 + 启动后端生产模式）
- 启动脚本输出局域网 IP 地址，方便用户获取访问 URL

**Non-Goals:**
- 不实现 HTTPS / SSL 证书配置（局域网环境不强制）
- 不实现 CDN / 反向代理 / Nginx 配置（后续单独处理）
- 不修改 Docker Compose（基础设施配置已足够）
- 不实现端口映射 / UPnP（用户自行在路由器配置）

## Decisions

### 1. 开发模式下 Vite 绑定 0.0.0.0 + CORS 环境变量

**选择**：Vite `host: '0.0.0.0'` + Fastify CORS 从环境变量读取允许列表。

**理由**：
- `0.0.0.0` 使 Vite 绑定所有网络接口，无需逐设备配置
- CORS origins 通过环境变量配置，用户可添加多个来源（手机 IP、平板 IP 等）
- 不设置 `*` 通配符 + `credentials: true`（浏览器安全策略禁止此组合）
- 默认值保留 `http://localhost:5173` 确保不改动本地开发体验

**未选择方案**：固定 IP 地址。每台机器 IP 不同，需要用户每次手动修改代码。

### 2. 生产模式：后端托管前端静态文件

**选择**：NestJS + Fastify 的 `@fastify/static` 提供静态文件服务，生产模式构建前端后由 NestJS 统一托管在单端口。

**理由**：
- 单一端口避免 CORS 问题
- 减少对外暴露的端口
- 与现有 Fastify 实例复用
- 前端构建产物约 2-3MB，内存占用可忽略

**未选择方案**：Nginx 反向代理。增加运维复杂度，当前阶段不必要。

### 3. 生产启动脚本

**选择**：新增 `start-prod.sh` / `.ps1` / `.bat`，自动构建前端 + 启动后端 prod 模式。

**理由**：
- 与现有 dev 脚本职责分离
- 脚本自动完成构建步骤，减少手动操作
- 控制台输出局域网 IP 地址

### 4. 局域网 IP 检测

**选择**：启动脚本中通过平台特定方式获取局域网 IP 并打印。

**理由**：
- Bash: `hostname -I` / `ipconfig` 获取网卡 IP
- PowerShell: `Get-NetIPAddress` 获取 IPv4 地址
- 打印 IP 地址方便用户在其他设备输入 URL

## Risks / Trade-offs

- [0.0.0.0 安全风险] 绑定所有接口意味着同一局域网内的任何设备都可以访问服务。→ 仅供开发和内部使用，不暴露到公网。
- [Vite HMR over network] Vite HMR 默认通过 WebSocket 连接，跨设备访问时 HMR 可能不可用。→ 影响可接受，跨设备访问主要用于测试完整功能而非热更新开发。
- [静态文件路径] 生产模式下后端需要知道前端 dist 路径。→ 使用 `path.join(__dirname, '../../client/dist')` 相对路径，构建输出目录固定。
