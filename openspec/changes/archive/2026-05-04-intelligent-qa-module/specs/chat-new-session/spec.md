## ADDED Requirements

### Requirement: 用户可以创建新的对话会话
The system SHALL 允许已登录用户创建具有唯一会话ID的新对话会话，并清除任何正在进行的流式回复。

#### Scenario: 成功创建新会话
- **WHEN** 用户点击"新建对话"按钮
- **THEN** 系统创建具有唯一UUID的新会话，停止任何正在进行的AI流式回复，清空聊天区域，展示欢迎语和推荐问题

#### Scenario: AI正在回复时创建新会话
- **WHEN** 用户在AI正在流式回复时点击"新建对话"
- **THEN** 系统通过AbortController中止当前流，丢弃未完成的AI消息，创建新会话

#### Scenario: 创建会话时网络失败
- **WHEN** 用户点击"新建对话"且网络请求失败
- **THEN** 系统展示错误提示"创建新对话失败，请检查网络或稍后重试"，用户停留在当前页面状态

### Requirement: 会话列表按更新时间排序返回
The system SHALL 按`updated_at`降序返回当前用户的会话列表，每条包含标题、最近更新时间和消息数量。

#### Scenario: 用户有多个会话
- **WHEN** 用户进入聊天页面
- **THEN** 系统按最近更新排序展示会话列表，显示标题（首个问题截取前30字或"新对话"）、更新时间和消息数量

#### Scenario: 用户没有会话
- **WHEN** 用户进入聊天页面且无聊天历史
- **THEN** 系统展示空状态，引导用户开始新对话
