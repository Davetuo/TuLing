## Context

途灵前端采用 Vue 3 + TypeScript + Vite + Vue Router + Pinia + Element Plus。当前 HomePage.vue 已有静态五大功能卡片（智能问答、景点探索、景点评价、行程规划、图片纪念墙），但缺少跳转逻辑、快捷入口区和状态处理。AppLayout.vue 已提供顶部导航栏（Logo、用户昵称、退出登录）。

技术约束：路由懒加载、Element Plus 按需引入、响应式布局支持 PC 和移动端、首页首屏只加载必要数据（异步加载快捷区内容）。

## Goals / Non-Goals

**Goals:**
- 功能入口卡片点击跳转至对应模块路由页面（初期为占位组件）
- 快捷入口区展示热门目的地（静态默认数据 + 预留 API 接口）、最近行程、最近对话
- 首页加载骨架屏、网络异常提示、各区块空状态占位
- 移动端响应式布局：卡片在小屏幕上一行两列
- 路由页面为占位组件，标记模块名称，后续模块实现时替换

**Non-Goals:**
- 不实现各功能模块的实际内容（智能问答、景点搜索等由后续 M3/M4/M5 交付）
- 不接入真实热门目的地 API（使用静态默认数据，预留接口类型和调用位置）
- 不实现最近行程/最近对话的真实数据拉取（后端对应接口尚未实现，展示空状态）
- 不改变现有鉴权逻辑和 AppLayout 结构

## Decisions

### 1. 功能入口跳转：Vue Router 命名路由 + 占位组件

每个功能入口卡片通过 `router.push({ name: 'Chat' })` 跳转。路由页面初期为最小占位组件（展示模块名称 + "即将上线"提示），后续模块实现时原地替换。

**Why:** 跳转链路先建立，后续模块开发互不阻塞。占位组件比空白页更友好，也便于 E2E 测试验证跳转逻辑。

**Alternatives considered:** 直接在卡片上放 `<router-link>` 语义更准确，但需要将 `el-card` 改造为 `<a>` 包裹，样式处理更复杂。采用 `@click` + `router.push` 更灵活，可支持未来埋点（`home_entry_click` 事件）。

### 2. 路由结构：四个新路由 + requiresAuth 守卫

在 `router/index.ts` 中新增：
- `/chat` → `pages/chat/ChatPage.vue`（智能问答）
- `/spots` → `pages/spots/SpotListPage.vue`（景点探索）
- `/trips` → `pages/trips/TripListPage.vue`（行程规划）
- `/albums` → `pages/albums/AlbumListPage.vue`（图片纪念墙）

所有路由均设置 `meta: { requiresAuth: true }`，复用现有路由守卫。

**Why:** 路由命名与 PRD 接口草案路径风格一致。每个模块独立目录，后续扩展子页面（如详情页、新建页）时目录结构清晰。

### 3. 快捷入口区：异步数据 + 降级展示

热门目的地使用静态默认数据（如北京、杭州、成都、厦门等 4-6 个城市），以卡片或标签形式展示，点击可跳转至景点探索页并预填城市参数。

最近行程和最近对话：调用后端 API 获取（使用 `onMounted` 异步请求），但当前后端接口未实现，前端先展示空状态（"暂无最近行程"/"暂无最近对话"）。

**Why:** 热门目的地用静态数据可立即交付首页体验，不阻塞后端开发。最近行程/对话的 API 调用预留好调用位置和类型，接口就绪后只需替换 API 方法即可。

### 4. 状态处理：骨架屏 + ElEmpty + ElAlert

- **加载中:** 使用 Element Plus 的 `<el-skeleton>` 模拟卡片骨架（3-5 行），页面初始化时展示
- **加载失败:** 使用 `<el-alert type="error">` 提示异常 + 重试按钮
- **空状态:** 使用 `<el-empty>` 展示（如无最近行程、无最近对话）
- **网络离线:** 监听 `navigator.onLine` + `window.addEventListener('offline'/'online')`，离线时顶部展示提示条

**Why:** PRD 13.1 要求"所有列表页需支持空状态、加载状态、失败状态"，骨架屏比全局 Loading 更符合首页场景（用户已有部分内容可见），ElEmpty/ElAlert 是 Element Plus 原生组件，无需自研。

### 5. 移动端响应式：Element Plus Grid + CSS Media Queries

- 功能卡片：`el-col` 响应式断点 `:xs="12" :sm="8"`（移动端两列，平板及以上三列），首页最大宽度从固定 960px 改为 `max-width: 1200px` 以容纳三列卡片
- 快捷入口区：移动端堆叠排列（flex-direction: column）
- 英雄区文字：移动端缩小字号
- 顶部导航：移动端隐藏副标题"AI 智能旅行搭子"，仅保留 Logo

**Why:** PRD 6.3.4 要求"首页在 PC 和移动浏览器上布局不变形"。Element Plus Grid 已提供断点能力，无需自定义媒体查询来处理响应式；顶部导航和英雄区的细节调整用少量 CSS 覆盖即可。

## Risks / Trade-offs

- **占位页面体验感弱** → 每个占位页展示模块名称 + 图标 + "功能开发中，敬请期待"文案，降低用户困惑
- **快捷区数据依赖后端接口** → 热门目的地使用静态数据；最近行程/对话在 API 不可用时展示空状态，不报错
- **移动端五列变两列导致首屏变长** → 卡片设计紧凑，两列布局下仍可在 1.5 屏内展示全部入口
- **骨架屏与真实卡片布局差异导致闪烁** → 骨架屏使用与真实卡片相同的网格布局（`el-row` + `el-col`），仅内容替换为 `<el-skeleton-item>` 占位块
